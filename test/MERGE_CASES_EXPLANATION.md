# ケース1、2、5の違いの説明

## 重要なポイント

**現在の実装では、ログイン時に「ローカルの`lastUpdated`」と「Firestoreの`lastUpdated`」を比較して、新しい方のデータ全体を採用します。**

## 3つのケースの違い

### ケース1: 別アカウントでログイン → 消える

**状況**:
```
時刻10:00 - アカウント1でログイン → 問題Aに「解けた」を登録
         → Firestore（アカウント1）に保存: lastUpdated = 10:00

時刻11:00 - ログアウト

時刻11:00 - ログアウト状態で問題Aに「理解できた」を登録
         → ローカルのみに保存: lastUpdated = 11:00

時刻12:00 - アカウント2でログイン
         → Firestore（アカウント2）からデータを取得: lastUpdated = 12:00
         （アカウント2の既存データ、または別デバイスで編集済み）
```

**比較**:
- ローカル: `lastUpdated = 11:00`（ログアウト後に登録した「理解できた」）
- Firestore（アカウント2）: `lastUpdated = 12:00`（アカウント2のデータ）

**結果**: Firestore（12:00）が新しい → Firestoreのデータを採用 → **ローカルの「理解できた」が消える**

**なぜ消える？**: アカウント2のFirestoreデータと比較するため、アカウント2のデータが新しいとローカルの編集が上書きされる

---

### ケース2: 同じアカウントだが別デバイスで編集 → 消える

**状況**:
```
時刻10:00 - デバイスAでアカウント1にログイン → 問題Aに「解けた」を登録
         → Firestoreに保存: lastUpdated = 10:00

時刻11:00 - デバイスAでログアウト

時刻11:00 - デバイスAでログアウト状態で問題Aに「理解できた」を登録
         → ローカル（デバイスA）のみに保存: lastUpdated = 11:00

時刻12:00 - デバイスBでアカウント1にログイン → 問題Aに「できなかった」を登録
         → Firestoreに保存: lastUpdated = 12:00

時刻13:00 - デバイスAでアカウント1に再ログイン
         → Firestoreからデータを取得: lastUpdated = 12:00
```

**比較**:
- ローカル（デバイスA）: `lastUpdated = 11:00`（ログアウト後に登録した「理解できた」）
- Firestore: `lastUpdated = 12:00`（デバイスBで登録した「できなかった」）

**結果**: Firestore（12:00）が新しい → Firestoreのデータを採用 → **ローカルの「理解できた」が消える**

**なぜ消える？**: 別デバイス（デバイスB）でFirestoreのデータが更新されたため、Firestoreが新しいとローカルの編集が上書きされる

---

### ケース5: 同じアカウントで別デバイスで編集なし → 保持される

**状況**:
```
時刻10:00 - アカウント1でログイン → 問題Aに「解けた」を登録
         → Firestoreに保存: lastUpdated = 10:00

時刻11:00 - ログアウト

時刻11:00 - ログアウト状態で問題Aに「理解できた」を登録
         → ローカルのみに保存: lastUpdated = 11:00

時刻12:00 - アカウント1に再ログイン（別デバイスで編集なし）
         → Firestoreからデータを取得: lastUpdated = 10:00（変更なし）
```

**比較**:
- ローカル: `lastUpdated = 11:00`（ログアウト後に登録した「理解できた」）
- Firestore: `lastUpdated = 10:00`（変更なし）

**結果**: ローカル（11:00）が新しい → ローカルのデータを採用 → **「理解できた」が保持される**

**なぜ保持される？**: 別デバイスで編集がなかったため、Firestoreのデータが古いまま。ローカルの編集が新しいので、ローカルのデータが採用される

---

## まとめ

| ケース | ログイン先 | 別デバイスで編集 | 結果 |
|--------|----------|----------------|------|
| ケース1 | 別アカウント | あり（アカウント2のデータが新しい） | ❌ 消える |
| ケース2 | 同じアカウント | あり（デバイスBで編集） | ❌ 消える |
| ケース5 | 同じアカウント | なし | ✅ 保持される |

**重要なポイント**:
- 消える理由は「Firestoreのデータがローカルより新しい」から
- ケース1: 別アカウントのFirestoreデータが新しい
- ケース2: 同じアカウントだが、別デバイスでFirestoreが更新された
- ケース5: 同じアカウントで、別デバイスで編集なし → Firestoreが古いまま

## ユーザーの期待との違い

ユーザーは「アカウント1ログアウトしてローカルでいじってアカウント1に入ろうがアカウント2に入ろうが同じ」と言っていました。

**期待**: ログアウト後のローカル編集は、どのアカウントにログインしても保持される

**現実**: 
- 同じアカウントに再ログイン → 別デバイスで編集がなければ保持される（ケース5）
- 同じアカウントに再ログイン → 別デバイスで編集があれば消える（ケース2）
- 別アカウントにログイン → そのアカウントのデータが新しいと消える（ケース1）

**問題点**: ケース1とケース2では、ログアウト後のローカル編集が失われる可能性がある
