# テストが検知できる異常ケース一覧

## 1. データ破損・不正データの検知

### 1.1 JSON破損データ
**検知できる異常:**
```dart
// テスト: handles corrupted JSON data
'joymath_simple/learning/problem_1': 'invalid-json'
```
**検知内容:**
- SharedPreferencesに保存されたJSONが破損している
- JSONパースエラーが発生する
- アプリがクラッシュする可能性

**期待される動作:**
- エラーをキャッチして、デフォルト値（`LearningStatus.none`）を返す
- アプリがクラッシュしない

### 1.2 無効なステータス値
**検知できる異常:**
```dart
// テスト: handles invalid status values
'latestStatus': 'invalid_status'  // 存在しないステータス
```
**検知内容:**
- データベースに存在しないステータス値が保存されている
- enum変換に失敗する可能性
- 予期しない動作が発生する可能性

**期待される動作:**
- 無効なステータスを`none`として扱う
- エラーを発生させない

### 1.3 破損したガチャ設定
**検知できる異常:**
```dart
// テスト: handles corrupted gacha settings gracefully
'joymath_simple/gacha/test_gacha': 'invalid-json'
```
**検知内容:**
- ガチャ設定のJSONが破損している
- フィルタモードが読み込めない
- ガチャが動作しない

**期待される動作:**
- デフォルト設定（`exclude_solved_ge1`）を返す
- ガチャが正常に動作し続ける

## 2. データ欠損・不完全データの検知

### 2.1 タイムスタンプの欠損
**検知できる異常:**
```dart
// テスト: handles missing time field
{'status': 'solved'}  // timeフィールドがない
```
**検知内容:**
- 学習履歴にタイムスタンプがない
- 時系列ソートができない
- 最新記録の判定ができない

**期待される動作:**
- `time: null`として処理する
- ステータスは正常に取得できる

### 2.2 無効なタイムフォーマット
**検知できる異常:**
```dart
// テスト: handles invalid time format
{'status': 'solved', 'time': 'invalid-time'}
```
**検知内容:**
- ISO8601形式でないタイムスタンプ
- `DateTime.parse()`が失敗する
- 例外が発生する可能性

**期待される動作:**
- パースエラーをキャッチして`null`を設定
- ステータスは正常に取得できる

### 2.3 null値の処理
**検知できる異常:**
```dart
// テスト: handles null time values

```
**検知内容:**
- タイムスタンプがnull
- null参照エラーの可能性
- ソート処理でエラー

**期待される動作:**
- nullを安全に処理する
- エラーを発生させない

## まとめ

これらのテストにより、以下の異常を検知できます：

1. **データ破損**: JSON破損、無効な値
2. **データ欠損**: フィールド欠損、null値
3. **ネットワーク異常**: タイムアウト、接続失敗
4. **データ整合性**: 上限超過、混在、影響範囲
5. **境界値**: 空データ、範囲外アクセス
6. **並行処理**: 競合状態、データ損失
7. **初期化**: 重複実行、設定更新

これらの異常が発生しても、アプリがクラッシュせず、適切にフォールバック処理が行われることを確認しています。
